# PHP 8 Compatibility Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make PHPDeobfuscator deprecation-clean on the latest PHP 8.x, declare PHP 8.0 as the minimum version, and update infrastructure (Dockerfile, flysystem) accordingly.

**Architecture:** This is a compatibility-fix plan — no architectural changes. Eight tasks each target one deprecation/warning category from the design spec, plus an infra-bump task and a final verification task. The existing `test.php` (30 fixture-driven tests) is the regression safety net at every step.

**Tech Stack:** PHP 8.x, Composer, `nikic/php-parser` v4, `league/flysystem` v3 (after upgrade).

**Verification command** (used at the end of every task):

```sh
php -d error_reporting=E_ALL test.php > /tmp/out 2> /tmp/err
grep -E "^(Deprecated|Warning|Fatal|PHP (Deprecated|Warning|Fatal))" /tmp/err | wc -l
grep -c "pass$" /tmp/out
grep -c "failed:" /tmp/out
```

The end-state targets are: deprecations `0`, passes `30`, fails `0`. Tasks 1–8 may temporarily leave non-zero deprecation counts (we fix categories incrementally), but **passes must remain `30` and fails must remain `0` after every task**.

## Task 4: Replace deprecated `ReflectionParameter::getClass()` (Group B)

**Files:**
- Modify: `src/Reducer/AbstractReducer.php:24-25`

**Background.** `ReflectionParameter::getClass()` was deprecated in PHP 8.0. The replacement is `getType()`, which returns a `ReflectionType`. For the single-class-typehint case used here (every reducer method takes one `Node`-implementing class), the type will be a `ReflectionNamedType` and we resolve it back to a `ReflectionClass` via `new \ReflectionClass($type->getName())`. The downstream code (`$type->implementsInterface(...)`, `$type->name`) operates on a `ReflectionClass`, which we preserve.

- [ ] **Step 1: Replace the `getClass()` call**

In `src/Reducer/AbstractReducer.php`, replace the existing line 24:

```php
                    $type = $param->getClass();
```

with:

```php
                    $paramType = $param->getType();
                    if (!($paramType instanceof \ReflectionNamedType) || $paramType->isBuiltin()) {
                        throw new \LogicException("Parameter must be a class type-hint");
                    }
                    $type = new \ReflectionClass($paramType->getName());
```

The downstream uses (`$type->implementsInterface(Node::class)` on line 25, `$type->name` on lines 28 and 29, `$this->methodMapping[$type->name]` on line 31) all work unchanged on a `\ReflectionClass`.

- [ ] **Step 2: Verify**

Run the full verification command.

Expected: passes = `30`, fails = `0`. The many repeated `Method ReflectionParameter::getClass() is deprecated` lines disappear. Deprecation count drops correspondingly (this is the loudest single deprecation in the baseline, fired once per reducer method).

- [ ] **Step 3: Commit**

```sh
git add src/Reducer/AbstractReducer.php
git commit -m "Replace deprecated ReflectionParameter::getClass() with getType()"
```

## Task 5: Declare properties to fix dynamic property creation (Group C)

**Files:**
- Modify: `src/Deobfuscator.php` (declare `$metaVisitor`)
- Modify: `src/EvalBlock.php` (declare `$origStmts`)
- Modify: `src/VarRef/ListVarRef.php` (declare `$vars`)
- Modify: `src/ControlFlowVisitor.php` (declare `$node`, `$subNodes`, `$stmts`, `$elseifs`, `$else` on `WrappedNode`)

**Background.** PHP 8.2 deprecated implicit creation of properties via assignment. The fix is to declare each property on the class. Per the design spec's targeted-modernization scope, declarations match the existing untyped style of each file (the surrounding class properties are untyped), except where the type is unambiguous from the constructor signature.

- [ ] **Step 1: Declare `$metaVisitor` on `Deobfuscator`**

In `src/Deobfuscator.php`, the existing private-property block reads:

```php
    private $parser;
    private $prettyPrinter;

    private $firstPass;
    private $secondPass;

    private $fileSystem;
    private $filename;
```

Add `$metaVisitor` to the same block (matching the file's untyped style):

```php
    private $parser;
    private $prettyPrinter;

    private $firstPass;
    private $secondPass;

    private $fileSystem;
    private $filename;

    private $metaVisitor;
```

- [ ] **Step 2: Declare `$origStmts` on `EvalBlock`**

In `src/EvalBlock.php`, change the property block to:

```php
class EvalBlock extends Expr
{
    public $stmts;
    public $origStmts;
```

(Add `public $origStmts;` immediately after the existing `public $stmts;`.)

- [ ] **Step 3: Declare `$vars` on `ListVarRef`**

In `src/VarRef/ListVarRef.php`, add a property declaration before the constructor. The constructor type-hints `array $vars`, so the type is unambiguous; declare it as `private array $vars;`. The class becomes:

```php
class ListVarRef implements VarRef
{
    private array $vars;

    public function __construct(array $vars)
    {
        $this->vars = $vars;
    }
```

- [ ] **Step 4: Declare the five properties on `WrappedNode`**

In `src/ControlFlowVisitor.php`, the `WrappedNode` class currently begins:

```php
class WrappedNode implements Node
{
    public function __construct(Node $node, array $subNodes)
    {
        $this->node = $node;
        $this->subNodes = $subNodes;
        foreach ($subNodes as $name) {
            $this->$name = $node->$name;
        }
    }
```

The constructor assigns `$node` (a `Node`), `$subNodes` (an `array`), and then dynamically copies the properties named in `$subNodes` from the source `$node`. The deprecation lists `$stmts`, `$elseifs`, `$else` — these are the fields that get dynamically copied for the `if` statement that the visitor wraps.

Insert a property declaration block before the constructor:

```php
class WrappedNode implements Node
{
    private Node $node;
    private array $subNodes;
    public $stmts;
    public $elseifs;
    public $else;

    public function __construct(Node $node, array $subNodes)
    {
        $this->node = $node;
        $this->subNodes = $subNodes;
        foreach ($subNodes as $name) {
            $this->$name = $node->$name;
        }
    }
```

The dynamic `$this->$name = $node->$name` loop now writes into the declared `$stmts` / `$elseifs` / `$else` properties. If a future call wraps a node with subNode names outside this set, that would re-trigger the deprecation — that is acceptable: the spec scopes us to fixing the deprecations the test suite emits today, not to a defensive sweep.

- [ ] **Step 5: Verify**

Run the full verification command.

Expected: passes = `30`, fails = `0`. All `Creation of dynamic property` lines are gone from stderr. Deprecation count drops by 8 (one per field listed in the baseline catalog).


## Task 6: Fix the smaller code-level deprecations and warnings (Group D)

**Files:**
- Modify: `src/Reducer/UnaryReducer.php:36` (non-canonical `(double)` cast)
- Modify: `src/Reducer/UnaryReducer.php:77` (string increment)
- Modify: `src/ValRef/ArrayVal.php:33` (null array offset)
- Modify: `src/Reducer/MiscReducer.php:19` (array-to-string warning)

- [ ] **Step 1: Replace `(double)` cast with `(float)`**

In `src/Reducer/UnaryReducer.php`, change line 36 from:

```php
        return Utils::scalarToNode((double) $val);
```

to:

```php
        return Utils::scalarToNode((float) $val);
```

`(double)` and `(float)` are aliases at runtime. The deprecation specifically targets the `(double)` spelling.

- [ ] **Step 2: Fix the non-numeric string increment in `postIncDec`**

In `src/Reducer/UnaryReducer.php`, line 77 currently reads:

```php
        $isInc ? $val++ : $val--;
```

PHP 8.3 deprecated `++` on strings that aren't numeric (the message: "Increment on non-numeric string is deprecated, use str_increment() instead"). The `--` case for non-numeric strings is historically a no-op and is **not** the deprecation we're fixing here, so we leave the decrement path alone. Replace line 77 with:

```php
        if ($isInc && is_string($val) && !is_numeric($val) && $val !== '') {
            $val = str_increment($val);
        } else {
            $isInc ? $val++ : $val--;
        }
```

The non-string path is unchanged. The string path uses `str_increment` only for the increment + non-empty + non-numeric-string case the deprecation flags. We deliberately do not switch `--` to `str_decrement`, because `str_decrement` is new behavior (it actually decrements alphabetic strings) whereas `--` was historically a no-op for them — using `str_decrement` would change semantics on inputs the test suite may rely on, and the deprecation we're targeting here only covers `++`.

- [ ] **Step 3: Fix the null array offset in `ArrayVal::arrayFetch`**

In `src/ValRef/ArrayVal.php`, the existing `arrayFetch` method (lines 30–37) reads:

```php
    public function arrayFetch($dim)
    {
        $this->checkMutable();
        if (!isset($this->backingArray()[$dim])) {
            return null;
        }
        return $this->backingArray()[$dim];
    }
```

Replace it with:

```php
    public function arrayFetch($dim)
    {
        $this->checkMutable();
        if ($dim === null) {
            $dim = '';
        }
        if (!isset($this->backingArray()[$dim])) {
            return null;
        }
        return $this->backingArray()[$dim];
    }
```

This preserves PHP's documented null-as-empty-string array-key coercion without triggering the deprecation.

- [ ] **Step 4: Fix the array-to-string warning in `MiscReducer::reduceEncapsedString`**

In `src/Reducer/MiscReducer.php`, the existing `reduceEncapsedString` body (lines 11–26) reads:

```php
    public function reduceEncapsedString(Node\Scalar\Encapsed $node)
    {
        $newString = '';
        foreach ($node->parts as $part) {
            if ($part instanceof Node\Scalar\EncapsedStringPart) {
                $newString .= $part->value;
            } else {
                try {
                    $newString .= Utils::getValue($part);
                } catch (\InvalidArgumentException $e) {
                    return null;
                }
            }
        }
        return Utils::scalarToNode($newString);
    }
```

If `Utils::getValue($part)` returns an array, the `.=` triggers the array-to-string conversion warning. The intended behavior when the value isn't string-convertible is the same as when it can't be evaluated — bail out and let the original AST node stand. Replace the inner `else` block with:

```php
            } else {
                try {
                    $value = Utils::getValue($part);
                } catch (\InvalidArgumentException $e) {
                    return null;
                }
                if (is_array($value) || is_object($value)) {
                    return null;
                }
                $newString .= $value;
            }
```

Objects without `__toString` would also warn, hence including `is_object` in the bail-out check.
